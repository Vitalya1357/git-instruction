## Введение

Git — это распределённая система контроля версий. Она позволяет сохранять историю изменений проекта, работать с ветками и объединять изменения.

GitHub — это онлайн-сервис для хранения Git-репозиториев и совместной работы над проектами.

## Создание учётной записи GitHub

Для создания учётной записи GitHub необходимо:

1. Перейти на сайт GitHub.

2. Нажать кнопку Sign up.

3. Ввести email.

4. Придумать пароль.

5. Указать имя пользователя.

6. Подтвердить регистрацию через email.

7. После входа в аккаунт создать новый репозиторий.

Пример названия репозитория:

```text
git-instruction
```

Локальный Git используется для работы с репозиторием на компьютере пользователя.

### Инициализация репозитория

Команда создаёт новый локальный Git-репозиторий.

Общий вид:

```bash
git init
```

### Проверка состояния репозитория

Команда показывает изменённые, добавленные и неотслеживаемые файлы.

Общий вид:

```bash
git status
```

### Добавление файла в индекс

Команда добавляет файл в область подготовленных изменений.

Общий вид:

```bash
git add имя_файла
```

Пример:

```bash
git add README.md
```

### Добавление всех файлов

Команда добавляет все изменённые файлы в индекс.

Команда показывает список подключённых удалённых репозиториев.

Общий вид:

```bash
git add .
```

### Создание коммита

Коммит фиксирует изменения в истории репозитория.

Общий вид:

```bash
git commit -m "Сообщение коммита"
```

Пример:

```bash
git commit -m "Добавлена инструкция по Git"
```

### Просмотр истории коммитов

Команда показывает историю изменений.

Общий вид:

```bash
git log
```

Пример:

```bash
git log --oneline
```

### Создание ветки

Команда создаёт новую ветку.

Общий вид:

```bash
git branch имя_ветки
```

Пример:

```bash
git branch local
```

### Переключение между ветками

Команда переключает пользователя на другую ветку.

Общий вид:

```bash
git switch имя_ветки
```

Пример: 

```bash
git switch local
```

### Слияние веток

Команда объединяет изменения из выбранной ветки с текущей веткой.

Общий вид:

```bash
git merge имя_ветки
```

Пример: 

```bash
git merge local
```

Сохрани файл.

Коммит в ветке `local`:

```bash
git add README.md
git commit -m "Добавлены команды локального Git"
```

## Работа с удалённым репозиторием

Удалённый репозиторий хранится на GitHub или другом Git-сервисе.

### Добавление удалённого репозитория

Команда связывает локальный репозиторий с удалённым.

Общий вид:

```bash
git remote add origin ссылка_на_репозиторий
```

Пример:

```bash
git remote add origin https://github.com/Vitalya1357/git-instruction.git
```

### Проверка удалённого репозитория

Команда показывает список подключённых удалённых репозиториев.

Общий вид:

```bash
git remote -v
```

Пример:

```bash
git remote -v
```

### Отправка изменений на GitHub

Команда отправляет локальные коммиты в удалённый репозиторий.

Общий вид:

```bash
git push origin имя_ветки
```

Пример:

```bash
git push origin master
```

### Первая отправка ветки

Команда отправляет ветку и связывает её с удалённой веткой.

Общий вид:

```bash
git push -u origin имя_ветки
```

Пример:

```bash
git push -u origin master
```

### Получение изменений из удалённого репозитория

Команда скачивает изменения и сразу объединяет их с текущей веткой.

Общий вид:

```bash
git pull origin имя_ветки
```

Пример:

```bash
git pull origin master
```

### Клонирование репозитория

Команда создаёт локальную копию удалённого репозитория.

Общий вид:

```bash
git clone ссылка_на_репозиторий
```

Пример:

```bash
git clone https://github.com/Vitalya1357/git-instruction.git
```

### Загрузка изменений без слияния

Команда скачивает информацию об изменениях, но не объединяет их автоматически.

Общий вид:

```bash
git fetch
```

Пример:

```bash
git fetch origin
```

## Работа с форками

Форк — это копия чужого репозитория в вашем аккаунте GitHub.

Форки используются, когда нужно предложить изменения в чужой проект.

### Принцип работы с форком

1. Открыть чужой репозиторий на GitHub.
2. Нажать кнопку Fork.
3. GitHub создаст копию репозитория в вашем аккаунте.
4. Склонировать форк на компьютер.
5. Создать отдельную ветку.
6. Внести изменения.
7. Отправить изменения в свой форк.
8. Создать Pull Request в исходный репозиторий.

### Клонирование форка

Команда создаёт локальную копию форка.

Общий вид:

```bash
git clone ссылка_на_форк
```

Пример:

```bash
git clone https://github.com/Vitalya1357/project-fork.git
```

### Добавление исходного репозитория

Команда добавляет оригинальный репозиторий как upstream.

Общий вид:

```bash
git remote add upstream ссылка_на_исходный_репозиторий
```

Пример:

```bash
git remote add upstream https://github.com/original-owner/project.git
```

### Получение изменений из исходного репозитория

Команда загружает изменения из оригинального репозитория.

Общий вид:

```bash
git fetch upstream
```

Пример:

```bash
git fetch upstream
```

### Обновление своей ветки из upstream

Команда объединяет изменения из исходного репозитория с текущей веткой.

Общий вид:

```bash
git merge upstream/master
```

Пример:

```bash
git merge upstream/master
```
